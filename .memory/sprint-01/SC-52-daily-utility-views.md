# SC-52 Delivery Memory: Daily Utility Views

## What Changed

### Backend
- Added `GET /api/v1/recommendations/today` — one-call "what should I buy today?" with top pick + alternatives + notable sales
- Added `is_watched` boolean field to `Merchant` model (SC-52 watch list)
- Added `GET /api/v1/merchants/watchlist` — list watched merchants
- Added `POST /api/v1/merchants/{id}/watch` — add to watchlist
- Added `DELETE /api/v1/merchants/{id}/watch` — remove from watchlist
- Updated `MerchantSummary` schema to expose `is_watched`

### Frontend
- New `/today` route + lazy component: Today buying brief with top pick, alternatives, and notable sales
- New `/watch` route + lazy component: Watchlist + Review Queue tabs
- New hooks: `use-today.ts`, `use-watchlist.ts`
- Updated sidebar nav: added Today (🗓️) and Watch & Review (👁️)
- Updated `routeTree.gen.ts` with `/today` (lazy) and `/watch` (lazy)

## Tricky Parts

- TanStack Router's `routeTree.gen.ts` is auto-generated by the Vite plugin but NOT by `tsc -b`. Solution: run `npx vite build` first to regenerate the route tree, then `tsc -b` passes cleanly. The final `npm run build` (which calls `tsc -b && vite build`) then works because the Vite step regenerates the tree before tsc would complain.
- The `/merchants/low-confidence` and `/merchants/watchlist` routes are static paths that must be ordered before `/{merchant_id}` in the FastAPI router — already handled correctly.
